# Stowage P1 Resource Baseline (D-126)

This file is generated by the profile rig (`make profile` /
`go test -tags=profile -run TestProfile ./internal/bench/profile/ -profile.write-baseline`).
It records the goroutine-stability and allocation-drift baselines from the
P1 profiling phase. Re-run `make profile` (with -profile.write-baseline) after any
change to the pipeline, lifecycle sweeps, or boot sequence to update these numbers.

**The goroutine-stability delta (S2-S0) is environment-independent.**
Absolute alloc/CPU numbers are local-machine-specific and listed here for
orientation only.

---

## Rig Description

The rig lives in `internal/bench/profile/` (build tag `profile`). Run it with:

```bash
make profile
# or with full options:
CGO_ENABLED=1 go test -tags=profile -v -run TestProfile ./internal/bench/profile/ \
  -profile.write-baseline \
  -profile.duration 5s \
  -profile.idle 3s \
  -profile.settle 2s \
  -profile.out /tmp/stowage-pprof
```

**What the rig does:**

1. **TestProfileIdle** — boots a full embedded Stowage instance (SQLite + mock
   gateway), waits for the idle window, samples goroutines and TotalAlloc before
   and after; drains. The goroutine delta must not climb (advisory gate).
2. **TestProfileLoad** — drives `-profile.ingest` + `-profile.retrieve` concurrent
   goroutines for `-profile.duration`, then drains and settles for `-profile.settle`.
   The post-drain goroutine count S2 is compared to the post-boot S0; the delta
   must be <= `-profile.eps`. Captures CPU / heap / goroutine / block / mutex pprof
   artifacts.
3. **TestProfileWriteBaseline** — collects the above results and writes this file.

**Flags:**

| Flag                     | Default | Description                                          |
|--------------------------|---------|------------------------------------------------------|
| -profile.ingest          | 16      | Concurrent ingest goroutines                         |
| -profile.retrieve        | 16      | Concurrent retrieve goroutines                       |
| -profile.duration        | 5s      | Load duration                                        |
| -profile.idle            | 3s      | Idle observation window                              |
| -profile.settle          | 2s      | Post-drain settle before final sample                |
| -profile.eps             | 50      | Allowed goroutine growth before stability gate fires |
| -profile.out             | ""      | pprof artifact dir; empty = t.TempDir() (ephemeral) |
| -profile.strict          | false   | Make gates fail the build (default: advisory log)   |
| -profile.write-baseline  | false   | (Re)write this file with measured numbers            |

## Capture Environment

- **Driver:** SQLite (pure-Go, CGo-free for the store; CGo enabled for -race)
- **Gateway:** mock (no network calls; deterministic embeddings)
- **Note:** Absolute alloc / CPU numbers are environment-specific (OS, Go
  runtime, machine). The goroutine-stability delta (S2-S0) is the primary
  signal and is environment-independent.

## Notes

- The **goroutine-stability gate** (S2 <= S0+eps) is the primary leak signal.
  "PASS" means all goroutines launched by the pipeline, sweeps, and lifecycle
  stages were collected after drain + settle.
- Errors during load are tolerated -- the mock gateway under concurrent load
  may return transient errors; the rig measures resources, not correctness.
- pprof artifacts are ephemeral by default (t.TempDir()). Set -profile.out
  to a persistent path and inspect with `go tool pprof`.
- The rig does NOT run under the default `go test ./...` (build tag `profile`
  guards it). It is a deliberate explicit gate (`make profile`), like `make slo`.
